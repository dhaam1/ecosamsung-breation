# 디자인 시안 제안 및 구현 가이드

사용자에게 디자인 시안을 제안하고 구현할 때 반드시 준수해야 하는 프로세스입니다.

## 1. 분석 및 파악 (Analysis)
- 사용자의 요구사항과 제공된 레퍼런스 이미지를 정밀 분석합니다.
- 분석 결과를 바탕으로 **3가지 추천 해결법**을 제시합니다.
  - **해결법 1**: 레퍼런스 충실형 (사용자 의도 100% 반영)
  - **해결법 2**: 트렌디/현대적 변주 (벤토 그리드, 글래스모피즘 등)
  - **해결법 3**: 파격적/에디토리얼 변주 (타이포그래피 강조, 독특한 레이아웃)
- 3가지 방법 중 **가장 성공 확률이 높은 방법**을 이유와 함께 추천합니다.

## 2. 시안 선택 인터페이스 (Switcher UI) 및 라벨링
- 사용자가 실시간으로 시안을 비교해볼 수 있도록 섹션 내부에 스위처 UI를 구현합니다.
- **섹션 라벨**: 모든 섹션 좌측 상단에 `Section 0X` 라벨을 표시합니다. (`absolute left-[5vw] top-[5vh] z-30`)
- **스위처 위치**: 해당 섹션의 우측 상단 (`absolute right-[5vw] top-[5vh] z-30`)
- **디자인**: 
  - 숫자 버튼 형태 (1, 2, 3)
  - 반투명 백드롭 블러 (`backdrop-blur-md`)와 은은한 테두리 (`border-white/10`) 적용
  - 활성화된 시안은 브랜드 컬러(`brand`)로 강조
- **상태 관리**: `useState`를 사용하여 `activeOption`을 관리합니다.

## 3. 애니메이션 및 전환 효과
- 시안 변경 시 사용자 경험을 극대화하기 위해 `AnimatePresence`를 사용합니다.
- 각 시안은 `initial`, `animate`, `exit` 속성을 통해 부드럽게 페이드 및 슬라이드되며 교체되어야 합니다.
- `mode="wait"`를 설정하여 이전 시안이 완전히 사라진 후 다음 시안이 나타나도록 합니다.

## 4. 최종 확정 프로세스
- 사용자가 특정 시안을 선택하면, 해당 시안의 코드만 남기고 스위처 UI와 나머지 시안 코드를 정리하여 최종 코드를 확정합니다.

---
**주의**: 이 가이드는 프로젝트의 심미적 완성도를 높이고 사용자의 선택권을 보장하기 위해 작성되었습니다. 모든 디자인 작업 시 이 프로세스를 기본으로 합니다.
